# study-planner-c

一个使用 C 语言编写的命令行学习任务管理工具。

当前版本：`v2.1`

## 项目简介

`study-planner-c` 用来在命令行中管理学习任务。它支持添加任务、查看任务、搜索任务、修改任务、删除任务、保存任务，并且会在下次启动时自动读取本地任务数据。

这个项目不使用第三方库，适合作为 C 语言基础练习项目，主要涉及：

- 结构体
- 数组
- 字符串处理
- 文件读写
- 菜单交互
- Windows 下 C 程序编译运行

## 当前功能列表

- 添加学习任务
- 设置任务优先级：`H / M / L`
- 设置任务截止日期，格式为 `MM-DD`
- 查看所有任务
- 按优先级和截止日期自动排序显示
- 按关键词搜索任务
- 搜索结果显示真实任务编号
- 查看或搜索后直接进入任务操作菜单
- 标记完成 / 取消完成
- 修改任务名称
- 修改任务优先级
- 修改任务截止日期
- 删除单个任务
- 清空全部任务
- 保存任务到本地文件
- 启动程序时自动读取任务
- 中文菜单和中文提示语
- 支持中文任务名称

## v1.8 已支持的功能

`v1.8` 重点增加了删除确认机制：

- 删除单个任务前需要确认
- 清空全部任务前需要确认
- 输入 `y` 或 `yes` 才会执行删除/清空
- 输入 `n` 或 `no` 会取消操作
- 输入其它内容会提示无效，并取消操作

这样可以避免误删单个任务或误清空全部任务。

## v2.0 代码结构重构

`v2.0` 不增加新功能，只整理代码结构。原来集中在 `src/main.c` 中的代码已经拆分为多个模块：

- `main.c`：程序入口
- `task.c / task.h`：任务结构、排序、优先级和日期校验
- `storage.c / storage.h`：任务文件保存和读取
- `ui.c / ui.h`：菜单、输入、搜索和任务操作流程

文件保存格式保持不变，仍然兼容已有的 `tasks.txt`。

## v2.1 稳定性测试和 bug 修复

`v2.1` 不增加新功能，只针对已有功能做稳定性检查和修复：

- 修复超长输入可能影响下一次菜单选择的问题
- 修复 `2abc` 这类非法编号可能被当成有效数字的问题
- 加强截止日期校验，例如 `02-31`、`04-31`、`13-01` 会被视为无效日期
- 重新测试添加、查看、搜索、修改、删除、清空、保存读取和中文任务输入

## 返回流程体验优化

本次优化不修改版本号，也不增加新功能，只调整查看和搜索后的返回体验：

- 在“查看所有任务”中进入任务操作菜单后，选择 `6. 返回` 会回到任务列表
- 在“搜索任务”中进入任务操作菜单后，选择 `6. 返回` 会回到搜索结果
- 在任务列表或搜索结果中输入 `0` 会直接返回主菜单
- 修改、标记完成、删除等操作完成后，会重新显示最新的任务列表或搜索结果

## Windows 下如何编译

推荐使用 Visual Studio 自带的 Microsoft C/C++ 编译器。

### 方法一：使用 VS 2022 开发者命令提示符

从开始菜单打开：

```text
x64 Native Tools Command Prompt for VS 2022
```

进入项目根目录：

```bat
cd /d "D:\Program Files\codex_program\first-program"
```

编译：

```bat
cl /utf-8 src\main.c src\task.c src\storage.c src\ui.c /Fe:study_planner.exe
```

### 方法二：在普通 CMD 中手动加载编译环境

如果使用普通 CMD，先执行：

```bat
call "D:\visual studio\VC\Auxiliary\Build\vcvars64.bat"
```

然后编译：

```bat
cd /d "D:\Program Files\codex_program\first-program"
cl /utf-8 src\main.c src\task.c src\storage.c src\ui.c /Fe:study_planner.exe
```

编译成功后会生成：

```text
study_planner.exe
```

## 如何运行

在项目根目录运行：

```bat
study_planner.exe
```

如果使用 PowerShell：

```powershell
.\study_planner.exe
```

## 使用示例

启动程序后会看到主菜单：

```text
==== 学习任务计划 ====
1. 添加任务
2. 查看所有任务
3. 搜索任务
4. 清空全部任务
5. 保存任务
6. 退出
请选择操作:
```

添加任务：

```text
请选择操作: 1
请输入任务名称: 复习 C 语言数组
优先级选项：1=高, 2=中, 3=低
请输入优先级: 1
请输入截止日期 (MM-DD): 06-20
任务已添加。
```

查看任务：

```text
请选择操作: 2

任务列表:
3. [ ] [H] [06-20] 复习 C 语言数组
2. [ ] [M] [06-18] 阅读教材
1. [ ] [L] [06-15] 整理笔记
```

任务前面的数字是真实编号，不是临时排序编号。后续操作都使用这个真实编号。

进入任务操作菜单：

```text
请输入要操作的任务编号，输入 0 返回: 3

任务:
3. [ ] [H] [06-20] 复习 C 语言数组

1. 标记完成 / 取消完成
2. 修改任务名称
3. 修改优先级
4. 修改截止日期
5. 删除任务
6. 返回
请选择操作:
```

搜索任务：

```text
请选择操作: 3
请输入搜索关键词: C

搜索结果:
3. [ ] [H] [06-20] 复习 C 语言数组
请输入要操作的任务编号，输入 0 返回: 3
```

删除单个任务前确认：

```text
请选择操作: 5
确认删除这个任务吗？输入 y/yes 确认，n/no 取消: n
删除已取消。
```

清空全部任务前确认：

```text
请选择操作: 4
确认清空全部任务吗？输入 y/yes 确认，n/no 取消: yes
全部任务已清空。
```

退出程序：

```text
请选择操作: 6
再见！
```

## 项目结构说明

```text
study-planner-c/
  .gitignore
  Makefile
  README.md
  src/
    main.c
    task.c
    task.h
    storage.c
    storage.h
    ui.c
    ui.h
```

文件说明：

```text
src/main.c      程序入口
src/task.c      任务结构、排序和校验相关代码
src/task.h      任务模块头文件
src/storage.c   任务保存和读取代码
src/storage.h   存储模块头文件
src/ui.c        菜单、输入和交互流程代码
src/ui.h        UI 模块头文件
Makefile        GCC/make 构建文件
README.md       项目说明文档
.gitignore      Git 忽略规则
```

运行后会生成：

```text
tasks.txt
```

`tasks.txt` 用来保存本地任务数据，不需要提交到 GitHub。

编译生成的 `.exe`、`.obj`、`.o` 等文件也不需要提交，已经在 `.gitignore` 中忽略。
